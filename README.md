# Q&A System — гибридная система поиска ответов

Интерактивная система семантического поиска нужного ответа по базе данных. 
Система поиска ответов на вопросы на основе гибридного подхода, объединяющего классический TF-IDF и современные эмбеддинги от Sentence Transformers. Проект предназначен для работы на CPU.

## 🧠 ML-подход

Проект использует **гибридный подход** для поиска наиболее релевантного ответа на вопрос пользователя. Это комбинация двух методов:

### 1. TF-IDF (Term Frequency — Inverse Document Frequency)
Классический статистический метод, который:
- Оценивает важность слов в вопросе относительно всей базы
- Хорошо работает для точных терминов (названия, технические термины)
- Не требует нейросетей, работает мгновенно

### 2. Sentence Transformers (семантические эмбеддинги)
Современный нейросетевой метод, который:
- Преобразует текст в плотные векторные представления (эмбеддинги)
- Понимает смысл, синонимы и перефразировки
- Использует предобученную мультиязычную модель для поддержки русского языка

### Гибридный метод
Сходство = 0.3 × TF-IDF_score + 0.7 × Embedding_score

text
Где:
- **TF-IDF_score** — насколько точно совпадают ключевые слова
- **Embedding_score** — насколько близки смыслы вопросов

### Почему именно этот подход?
- ✅ **Точность**: сочетает точность ключевых слов и понимание смысла
- ✅ **Производительность**: работает на CPU без необходимости в GPU
- ✅ **Гибкость**: можно переключаться между методами в реальном времени
- ✅ **Русский язык**: мультиязычная модель понимает русские вопросы

## ⚙️ Настройка окружения

### Файл `.env`

Создайте файл `.env` в корневой папке проекта:

```bash
# Модель эмбеддингов (поддерживает русский язык)
MODEL_NAME=paraphrase-multilingual-MiniLM-L12-v2

# TF-IDF параметры
TFIDF_MAX_FEATURES=5000

# Гибридные веса
HYBRID_WEIGHT_TFIDF=0.3
HYBRID_WEIGHT_EMBEDDING=0.7

# Порог сходства (0-1)
SIMILARITY_THRESHOLD=0.5

# Метод поиска по умолчанию
METHOD=hybrid

# Batch size для энкодинга
ENCODE_BATCH_SIZE=32

# Уровень логирования (DEBUG, INFO, WARNING, ERROR)
LOG_LEVEL=INFO

# Названия колонок в CSV
QUESTION_COL=вопрос
ANSWER_COL=ответ
```

### Очистка кэша при запуске (true/false)
| Параметр | Описание | Значения |
|-----------|-----------|-----------|
| `MODEL_NAME` | Модель для эмбеддингов | `paraphrase-multilingual-MiniLM-L12-v2` (русский), `all-MiniLM-L6-v2` (быстрая) |
| `TFIDF_MAX_FEATURES` | Максимум признаков для TF-IDF | `1000–10000` |
| `HYBRID_WEIGHT_TFIDF` | Вес TF-IDF в гибридном методе | `0.0–1.0` |
| `HYBRID_WEIGHT_EMBEDDING` | Вес эмбеддингов в гибридном методе | `0.0–1.0` |
| `SIMILARITY_THRESHOLD` | Минимальное сходство для выдачи ответа | `0.0–1.0` |
| `METHOD` | Метод поиска по умолчанию | `tfidf`, `embedding`, `hybrid` |
| `ENCODE_BATCH_SIZE` | Размер батча для энкодинга | `8–64` |
| `LOG_LEVEL` | Уровень детализации логов | `DEBUG`, `INFO`, `WARNING`, `ERROR` |
| `QUESTION_COL` | Название колонки с вопросами | Любое |
| `ANSWER_COL` | Название колонки с ответами | Любое |
| `NO_CACHE` | Очистка кэша при запуске | `true`, `false` |

## 🧩 Основные компоненты

QA_System (основной класс)
- Инициализация:
```python
qa = HybridQASystem(
    questions=questions,           # список вопросов
    answers=answers,               # список ответов
    model_name=MODEL_NAME,         # модель эмбеддингов
    threshold=0.5,                 # порог сходства
)
```

Методы:
- find_answer(query, method) — поиск ответа на вопрос
- get_stats() — статистика системы
- add_question_answer(question, answer) — добавление нового вопроса
- data_loader.py (загрузка данных)

Поддерживаемые форматы:
- .xlsx, .xls — через pandas + openpyxl
- .csv — с автоматическим определением кодировки

## 💾 Кэширование
Как работает кэширование
1. При первом запуске:
- Модель скачивается с Hugging Face
- Эмбеддинги вычисляются для всех вопросов
- Результат сохраняется в cache/embeddings_*.pkl

2. При последующих запусках:
- Эмбеддинги загружаются из кэша
- Модель загружается только при необходимости
- Запуск происходит мгновенно

3. Где хранится кэш
```text
cache/
└── embeddings_paraphrase-multilingual-MiniLM-L12-v2.pkl
```

Команды для управления кэшем
```bash
# Очистка кэша (при запуске)
python -m src.main --data data/questions.csv --no-cache

# Удаление кэша вручную
rm -rf cache/*

# Просмотр размера кэша
du -sh cache/

# Просмотр содержимого кэша (дата создания, размер)
ls -la cache/
```

Преимущества кэширования:
- Быстрый запуск — 0.5-1 секунда вместо 1-2 минут
- Экономия ресурсов — не нужно пересчитывать эмбеддинги
- Оффлайн работа — модель не скачивается при каждом запуске
